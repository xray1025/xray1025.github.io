Java集合

Author: X_ray Date: 十月 18, 2020  14:54:55 Category: Java

集合

  • Collection：List（列表）、Queue（队列）、Set（集）
  • Iterator（迭代器）

1 ArrayList和LinkedList的区别？

  • 是否线程安全：ArrayList和LinkedList都不是线程安全的；
  • 底层数据结构：ArrayList底层使用数组，LinkedList底层使用双向链表；
  • 快速随机访问：ArrayList支持，LinkedList不支持；
  • 随机位置插入删除的时间复杂度：ArrayList为O(n)，LinkedList为O(1);
  • 内存占用：ArrayList会在列表结尾预留一定容量空间，LinkedList的每个元素都要额外存储直接前驱和直接后继。

2 ArrayList和Vector的区别？

  • ArrayList不是线程安全的；
  • Vector是线程安全的，但是在同步操作上会耗费大量时间。

3 ArrayList的扩容机制？

  • 默认初始大小为10；

  • 底层其实是调用Arrays.copyOf`方法来进行扩充数组容量。

  • 默认情况下，新的容量会是原容量的1.5倍，这里用了位运算提高效率。一般情况下，如果扩容1.5倍后就大于期望容量，那就返回这个1.5倍旧容量的值。而如果小于期望容量，那就返回期望容量。

  • 使用1.5倍这个数值而不是直接使用期望容量，是为了防止频繁扩容影响性能。试想如果每次add操作都要扩容一次，那性能将会非常低下。

  • 可以手动扩容，没有缩容。

4 HashMap和HashTable的区别？

  • 线程是否安全：HashMap是非线程安全的，HashTable使用synchronized修饰，是线程安全的；
  • 效率：HashTable的效率低，已被淘汰；
  • Null值：HashMap的Key和Value可以为null，HashTab的Key如果为null，则会跑出NullPointerException。
  • 初始容量和扩容：HashMap的初始大小为16，扩容因子为0.75，每次扩容变为2倍，HashTable的初始大小为11，每次扩容变为原来的2n+1;
  • 底层实现不同。

5 HashMap和HashSet的区别？

  • HashSet的底层是基于HashMap实现的；

6 HashSet如何检查重复？

向HashSet添加元素时，HashSet会计算元素的hashCode判断元素加入的位置，同时与其他已添加的元素的hashCode作比较，如果没有hashCode相同的元素，则添加，否则会调用equals()方法检查hashCode相同的元素是否真的相同。

6.1 hashCode()和equals()

  • 如果两个对象相等，则hashCode相等，equals()为true；
  • 两个对象即使hashCode相等，也不一定相等；
  • 覆盖equals()必须覆盖hashCode()；
  • 如果没有重写hashCode()，则两个对象绝对不相等。

6.2 ==和equals()的区别？

  • 对于基本类型的变量来说（如 short、 int、 long、 float、 double），只能使用 == ，因为这些基本类型的变量没有 equals 方法。对于基本类型变量的比较，使用 == 比较， 一般比较的是它们的值。
  • 对于引用类型的变量来说（例如 String 类）才有 equals 方法。对于该类型对象的比较，默认情况下，也就是没有复写 Object 类的 equals 方法，使用 == 和 equals 比较是一样效果的，都是比较的是它们在内存中的存放地址。String 类存在 equals 方法被复写的情况，使用 equals 方法会比较它们的值；

7 HashMap的底层实现

  • JDK1.8以前，HashMap的底层是数组和链表组合在一起。HashMap计算key的hashCode经过扰动函数处理后得到hash值，然后通过 (n-1) & hash判断当前元素存放的位置，如果当前位置存在元素，则判断该元素与要存入的元素的hash值和key是否相等，如果相等则覆盖，否则通过拉链法解决冲突。

8 HashMap的长度为什么是2的幂次方？

hash % length == hash & (length - 1)的前提是length == 2 的幂次方。

9 ConcurrentHashMap和HashTable的区别？

  • 实现线程安全的方式不同：
    • JDK1.7的ConcurrentHashMap使用分段锁对整个数组进行分段加锁（Segment），多线程访问不同数据段的数据就不会发生锁竞争，提高并发效率；
    • JDK1.8的ConcurrentHashMap使用synchronized和CAS进行并发控制；
    • HashTable使用synchronized保证线程安全，效率低下。
  • 底层数据结构不同。

Segment：实现了ReentrantLock，是可重入锁。
